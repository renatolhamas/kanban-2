Quero criar um sistema de acompanhamento de conversas no Whatsapp no estilo Kanban (ou seja, uma pipeline visual de conversas com stages configuráveis).
Vamos utilizar “A VERSÃO 2” da Evolution API cuja documentação está no seguinte link: https://doc.evolution-api.com/v2/api-reference.
Penso que o sistema poderia ter:
- Página “Login” funcionando e devidamente conectada com o banco de dados. RLS funcionando, separando corretamente os dados de forma que cada usuário veja apenas os dados dele.
- Página “Register” funcionando e devidamente conectada com o banco de dados. RLS funcionando, separando corretamente os dados de forma que cada usuário veja apenas os dados dele.
- Página “Home”, com um Kanban, onde cada conversa vai ser um quadro (tipo um pequeno post-it). Para alterar as conversas vai arrastar o post-it. Quando aperta em cima do quadro abre o modal “conversa”, sendo possível enviar mensagens ou imagens. O usuário pode escrever uma mensagem ou escolher uma dentre as cadastradas em “Automatic Message”. Na parte superior do modal “conversa” o usuário pode escolher o “kanban” e a “coluna do kanban” para qual quer transferir a conversa (tipo uma caixa de seleção, com nome do kanban e nome da coluna: “kanban - coluna” em ordem alfabética de kanban e dentro do kanban na ordem da coluna ordem”).
- Página “Contacts”, com as colunas “Name”, “Phone Number” e Actions (em cada linha o ícone de um lápis, para editar o contato). Na parte superior direita, um botão NOVO, que se apertado abre o modal “Create Contact” com os campos “Name” e “Number” (formato internacional de telefone começando com “+”) e o botão “Save”. Criar também o modal “Edit Contact” com idêntico ao “Create Contact”. CRUD funcionando. Quando uma conversa é iniciada deve gravar o contato na tabela contacts.
- Página “Configurations” com os seguintes botões: “Profile”, “Connection”, “Automatic messages” e “Kanbans”.
-- Na página “Profile”, para alterar os dados do usuário logado (Nome e Senha).
-- Na página “Connection” é onde vai ocorrer as funcionalidades necessárias para conectar com o whatsapp conforme a documentação a API (Evolution API).
-- Na página “Automatic messages” as colunas “Name” e Actions (em cada linha o ícone de um lápis, para editar a mensagem automática). Na parte superior direita, um botão NOVO, que se apertado abre o modal “Create Message” com os campos “Name” e “Message” e o botão “Save”. Criar também o modal “Edit Message” com idêntico ao “Create Message”. CRUD funcionando. As mensagens serão configuráveis por kanban por período para ser como um follow up (ou poderão também ser enviadas se o usuário selecionar).
-- Na página “Kanbans” as colunas “Name”, “Main” (campo tipo “radio” onde pode marcar apenas um kanban da lista, que vai ser o kanban para o qual uma mensagem nova vai ser encaminhada para) e “Actions” (em cada linha o ícone de um lápis, para editar a mensagem automática). Na parte superior direita, um botão NOVO, que se apertado abre o modal “Create Kanban” com os campos “Name”, “Order” e os botões “Antes” e “Depois”. Name vai ser o nome da coluna do Kanban, Order vai ser a ordem quem que as colunas vão aparecer na página “home” sendo da esquerda para a direita o número do menor para o maior. Criar também o modal “Edit Kanban” com idêntico ao “Create Kanban”. CRUD funcionando.
Acima à direita, o ícone do usuário (que quando apertado vai para “profile”) e ao lado direito dele um botão vermelho com um “X” que quando apertado efetua o logoff.
