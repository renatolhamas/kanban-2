# Plano F4.007: Transição para o Padrão "Never Disable"

Este documento detalha o mapeamento e a estratégia de migração do padrão de comportamento de botões de formulário: do atual **"Bloqueio Preventivo"** (botão desabilitado até validação) para o padrão de alta performance **"Never Disable"** (botão sempre ativo, feedback pós-clique).

---

## 🧭 Visão Geral

A adoção do padrão "Never Disable" visa melhorar a descoberta de erros e a usabilidade do sistema, garantindo que o usuário nunca encontre um "caminho sem saída" visual.

---

## 🗺️ Mapeamento DE - PARA (Documentação e Código)

Abaixo, a lista exata de onde a "lei" do projeto está escrita e como ela deve ser atualizada.

| :--- | :--- | :--- |
| **`DESIGN.md`** | Seção 6: Do's and Don'ts | Adicionar Princípio #7: "Always Active Actions". Definir que botões de submit nunca ficam `disabled` por falta de input. |
| **`Button.md`** | Seção 2: Variants | Adicionar nota de UX: "O estado `disabled` deve ser usado apenas para estados de carregamento ou restrições de permissão, não para validação de formulário." |

| **`Story 2.4`** | AC #39 / Task 131 | Redefinir `variant="disabled"` apenas para casos de `loading` ou `locked`. |
| **Login Page** | `app/(auth)/login/page.tsx` | Remover `disabled={!isFormValid}`. O `handleSubmit` deve chamar `validateForm()` e abortar mostrando erros se necessário. |
