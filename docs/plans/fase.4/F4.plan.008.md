# F4.plan.008 — Auth-Aware Layout (Sidebar + UserMenu)

**Status:** PENDING  
**Data:** 2026-04-17  
**Prioridade:** CRÍTICA

---

## Contexto

A `Sidebar` e o `UserMenu` aparecem para qualquer visitante — mesmo sem estar logado.  
A lógica atual esconde esses elementos apenas em rotas de autenticação (`/login`, `/register`, etc.),  
mas **não verifica se há um usuário autenticado de fato**.

O resultado visível na screenshot: usuário não logado vê a Sidebar e o UserMenu na tela home.

## Solução

### Passo 1 — Criar `AuthContext` + `useAuth`

**Arquivo:** `src/context/AuthContext.tsx`

- Chama `GET /api/auth/me` ao montar (endpoint já existe e retorna `{ authenticated: bool, sub: string }`)
- Expõe `{ isAuthenticated, isLoading, user }` via context
- Enquanto carrega: `isLoading = true` (evita flash de conteúdo errado)

```tsx
// Contrato público
interface AuthContextValue {
  isAuthenticated: boolean;
  isLoading: boolean;
  user: { sub: string } | null;
}
```

**Arquivo:** `src/hooks/useAuth.ts`

```ts
export function useAuth(): AuthContextValue {
  return useContext(AuthContext);
}
```

## Bug Crítico Relacionado — Logout Quebrado

### Problema

O botão Logout em `UserMenu.tsx` (linha 61) faz:
```ts
localStorage.removeItem('auth_token');  // ← errado: token não está no localStorage
router.push('/login');
```

O token real é um **httpOnly cookie** chamado `auth_token` — não acessível via JavaScript/localStorage.  
A API `POST /api/auth/logout` (`app/api/auth/logout/route.ts`) existe e faz o correto (limpa o cookie),  
**mas nunca é chamada**.

### Consequência

1. Usuário clica Logout → localStorage limpo (sem efeito real), redireciona para `/login`
2. Cookie `auth_token` ainda existe → middleware detecta sessão ativa
3. Middleware redireciona de `/login` para `/` → parece que logou sozinho

### Fix (incluir no Passo 1)

**Arquivo:** `src/components/layout/UserMenu.tsx` — `handleLogout` (linha 58-66)

```ts
const handleLogout = async () => {
  try {
    await fetch('/api/auth/logout', { method: 'POST' });
    router.push('/login');
  } catch (error) {
    console.error('Logout failed:', error);
  }
};
```
