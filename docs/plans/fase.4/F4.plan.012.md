# Plano de Ação: F4.plan.012 [CONCLUÍDO] - Correção de Assinatura JWT & Otimização de Cliente

> [!NOTE]
> Este plano complementa o **F4.plan.011**, partindo da estrutura de diretórios organizada para focar na resolução técnica do erro de assinatura JWT e na performance do cliente Supabase.

## Contexto do Problema
O erro `PGRST301: None of the keys was able to decode the JWT` indica que o banco de dados Supabase (PostgREST) não conseguiu validar a assinatura do token enviado pelo frontend.
1. **Segredo Desatualizado:** O `JWT_SECRET` não está configurado no ambiente local, forçando o uso de uma chave de fallback que não coincide com a do Supabase.
2. **Excesso de Instâncias:** O sistema está criando novos clientes Supabase a cada chamada de hook, disparando alertas de instâncias duplicadas no browser.

**Objetivo:** Garantir que o `JWT_SECRET` real seja utilizado e otimizar a gestão de instâncias do cliente Supabase para performance e estabilidade.

---

## Passo a Passo para Resolução

### 1. Refatoração do Cliente Supabase (Singleton)
- **Arquivo:** `lib/supabase-client.ts`
- **Ação:** Implementar um mecanismo de cache em memória para armazenar as instâncias do cliente por token.
- **Benefício:** Elimina o aviso de "Multiple GoTrueClient instances" e economiza recursos do navegador.

### 2. Validação Rigorosa do JWT Secret
- **Arquivo:** `lib/jwt.ts`
- **Ação:** Atualizar `getJWTSecret()` para lançar um aviso mais visível (ou erro em produção) se o segredo for o padrão de desenvolvimento.
- **Ação:** Documentar claramente que o segredo deve ser o obtido no Dashboard do Supabase.

### 3. Configuração de Ambiente
- **Ação:** O desenvolvedor deve adicionar `JWT_SECRET` ao `.env.local`.

---

## User Review Required

> [!IMPORTANT]
> **AÇÃO NECESSÁRIA:** Você precisa acessar o seu **Dashboard do Supabase** > **Settings** > **API**.
> Localize o campo **JWT Secret**, copie o valor e adicione ao seu arquivo `.env.local`:
> ```env
> JWT_SECRET=seu-jwt-secret-do-supabase-aqui
> ```

---

## Verificação de Sucesso

### 1. Desaparecimento de Avisos
- O console do browser não deve mais mostrar "Multiple GoTrueClient instances detected".

### 2. Autorização RLS
- As requisições `GET /kanbans` devem retornar status `200 OK` e dados reais, evidenciando que a assinatura do JWT foi aceita pelo Supabase.

### 3. Persistência
- O fluxo de login e refresh (`/home`) deve manter o estado autenticado sem erros de decodificação de token.
