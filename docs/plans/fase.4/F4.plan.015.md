# Plano de Ação: F4.plan.015 [CONCLUÍDO] - Alinhamento com Supabase Auth Nativo

> [!NOTE]
> Este plano complementa o **F4.plan.014**, implementando as soluções técnicas para os problemas de assinatura de token identificados na fase de pesquisa.

## Contexto do Problema
O erro `PGRST301: None of the keys was able to decode the JWT` ocorria porque estávamos gerando tokens manuais (custom-signed) no backend Next.js que o gateway do Supabase Cloud não conseguia validar. Embora estivéssemos usando o `JWT_SECRET` do dashboard, discrepâncias de algoritmo (ES256 vs HS256) e de issuer impediam a autorização.

**Objetivo:** Abandonar a geração manual de tokens e utilizar o `access_token` oficial emitido pelo Supabase Auth.

---

## Mudanças Realizadas

### 1. Backend: Fluxo de Login
- **Arquivo:** `app/api/auth/login/route.ts`
- **Ação:** O login agora usa o `access_token` oficial retornado pela chamada `supabase.auth.signInWithPassword()`.
- **Ação:** Este token oficial é armazenado no cookie `auth_token` e compartilhado com o frontend.

### 2. Backend: Middleware e Verificação
- **Arquivo:** `lib/jwt.ts` e `lib/types.ts`
- **Ação:** Atualizada a função `verifyJWT` para mapear os claims do Supabase (`app_metadata.tenant_id`) para a estrutura interna do projeto.
- **Ação:** Removida a lógica de geração manual (`generateJWT`) para evitar uso acidental.

### 3. Frontend: Sincronização de Hooks
- **Arquivo:** `src/hooks/useKanbans.ts` e `src/hooks/useConversations.ts`
- **Ação:** Hooks atualizados para reagir à disponibilidade do token oficial e injetá-lo nas chamadas do Supabase Client.

---

## Verificação de Sucesso

### 1. Eliminação do PGRST301
- As chamadas para `/rest/v1/kanbans` agora retornam status `200 OK` utilizando o token oficial.

### 2. Manutenção do Multi-Tenancy
- O isolamento por `tenant_id` continua funcionando via RLS, pois o Supabase injeta o `tenant_id` no JWT via `app_metadata`.

### 3. Simplicidade Arquitetural
- O sistema não precisa mais gerenciar chaves de assinatura manualmente, confiando no motor nativo do Supabase.
