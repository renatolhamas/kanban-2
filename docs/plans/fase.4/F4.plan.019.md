# Plan 019: Fix Quality Gate Blockers & Technical Debt

**Objetivo:** Resolver os impedimentos do "Quality Gate" reportados pelo DevOps (Gage), garantindo que a build de produção passe e o repositório mantenha a integridade de tipos e padrões de código.

## 📋 Diagnóstico (Quality Gate Failures)

1. **TypeScript (`lib/supabase-client.ts`)**: Erro de atribuição no cache de clientes devido a incompatibilidade de tipos inferidos.
2. **Test Failures (`lib/jwt.ts`)**: `generateJWT` está comentado/desativado, quebrando os testes de integração do Evo GO QR.
3. **Lint Errors**: 2 instâncias de `Unexpected any` (`no-explicit-any`) em `lib/jwt.ts` e `lib/middleware/auth.ts`.
4. **Git Checks**: Diversos arquivos com *trailing whitespaces* (espaços ao final da linha).

---

## 🛠️ Mudanças Propostas

### 1. Correção do Supabase Client Factory
- **Arquivo:** `lib/supabase-client.ts`
- **Ação:** Tipar explicitamente o `clientCache` para aceitar a interface base do `SupabaseClient`.
- **Risco:** Baixo (ajuste de tipagem, não altera lógica de conexão).

### 2. Restauração/Mock de `generateJWT` para Testes
- **Arquivo:** `lib/jwt.ts`
- **Ação:** Re-habilitar a exportação da função `generateJWT` (necessária para os testes de integração).
- **Risco:** Baixo (função usada apenas em contextos controlados e testes).

### 3. Resolução de Erros de Lint (Explicit Any)
- **Arquivos:** `lib/jwt.ts`, `lib/middleware/auth.ts`
- **Ação:** Substituir o uso de `any` por interfaces específicas (`JWTPayload`).
- **Risco:** Baixo (melhora a segurança de tipos).

### 4. Limpeza de Espaços em Branco (Git Hygiene)
- **Ação:** Executar correção automática em todos os arquivos reportados pelo DevOps.
