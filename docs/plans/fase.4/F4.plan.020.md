# PLAN.020: Correção do Erro de Geração de QR Code WhatsApp [✅ CONCLUÍDO]


## Contexto
Ao tentar conectar o WhatsApp na página `/settings`, o sistema retorna "Internal Server Error" (500) e não exibe o QR Code.

## Causa Raiz
Identificamos uma inconsistência nas variáveis de ambiente Supabase nos drivers do Evo Go:
- `app/api/settings/evo-go/qr/route.ts` e `app/api/settings/evo-go/disconnect/route.ts` utilizam `process.env.SUPABASE_URL`.
- O arquivo `.env.local` define apenas `NEXT_PUBLIC_SUPABASE_URL`.
- O arquivo `.env` possui `SUPABASE_URL` vazio.

Como `SUPABASE_URL` está vazio/indefinido nessas rotas, o código retorna um erro de configuração (500).

## Plano de Ação

### 1. Correção das Rotas API
Atualizar as rotas para suportar `NEXT_PUBLIC_SUPABASE_URL` como fallback, garantindo compatibilidade com o ambiente de desenvolvimento local.

#### Arquivos afetados:
- `app/api/settings/evo-go/qr/route.ts`
- `app/api/settings/evo-go/disconnect/route.ts`

### 2. Padronização do `.env.local`
Adicionar `SUPABASE_URL` explicitamente no `.env.local` para evitar problemas com outros drivers puramente server-side.
