# F4.plan.022 - Padronização da Autenticação na Rota de Perfil

Este plano documenta a correção para o erro de autenticação (401) na página de perfil, alinhando sua lógica de validação de JWT com o padrão utilizado no restante da aplicação.

## Contexto e Problema

O usuário relatou que ao clicar em "Profile", a aplicação tenta carregar a página mas redireciona para `/home`. 

**Causa Raiz:**
- A rota `/api/auth/profile` utiliza uma validação de JWT manual e offline (`lib/jwt.ts`) baseada no segredo local `JWT_SECRET`.
- Os tokens emitidos pelo Supabase falham nessa validação se o segredo local não coincidir ou se houver discrepâncias na implementação.
- O restante do sistema usa `lib/middleware/auth.ts` que valida via `supabase.auth.getUser()`.

## Mudanças Propostas

### Backend (API)

#### [MODIFY] [route.ts](file:///c:/git/kanban.2/app/api/auth/profile/route.ts)
- Importar `auth` de `@/lib/middleware/auth`.
- Substituir a validação manual nos métodos `GET` e `PUT`.
- Remover imports não utilizados (`getJWTFromCookie`, `verifyJWT`).
