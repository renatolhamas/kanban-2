# Plano de Ação F4.plan.023 — Expansão do Kanban CRUD: Gestão de Colunas

**Autor:** Dex (AIOX Developer)
**Data:** 2026-04-29
**Story de Referência:** 4.4 (Extensão)

## 🎯 Objetivo
Ampliar as capacidades de gerenciamento de Kanbans, permitindo que o usuário gerencie (Criar, Editar, Excluir e Reordenar) as colunas de cada board de forma independente e segura.

## 🏗️ Arquitetura de UI: Página vs Modal
Após análise técnica, optamos pela transição para uma **Página de Configuração de Kanban** (`/settings/kanbans/[id]`).
- **Motivação:** Evitar modais aninhados e permitir uma interface de tabela limpa para as colunas, similar à listagem de Kanbans.
- **Fluxo:** Na aba "Kanbans" em Settings, o botão de edição (lápis) agora redirecionará para esta nova página.
- **Navegação de Retorno:** A página terá um botão "Voltar" que redirecionará para `/settings?tab=kanbans`, preservando o contexto do usuário.
- **Persistência:** 
    - Edição de nome do Kanban: Botão "Salvar" explícito no header.
    - Gestão de Colunas: Ações atômicas e instantâneas (Reorder/Delete/Rename) para melhor UX.

## 🛠️ Especificações Técnicas

### 1. Novas Rotas e Componentes
- **[NEW]** `app/settings/kanbans/[id]/page.tsx`: Página principal de gestão do Kanban.
- **[NEW]** `src/components/ui/organisms/settings/ColumnTable.tsx`: Tabela para listagem e ações de colunas.
- **[NEW]** `src/components/ui/organisms/settings/ColumnModals.tsx`: Modais para criar/editar colunas.

### 2. API Endpoints (Application-Layer Orchestration)
- `GET /api/kanbans/[id]/columns`: Busca colunas de um Kanban específico.
- `POST /api/columns`: Cria nova coluna (limite de 10 por Kanban).
- `PUT /api/columns/[id]`: Altera nome da coluna.
- `DELETE /api/columns/[id]`: 
    - **Orquestração:** Move conversas da coluna deletada para a coluna anterior (ou a primeira disponível) do mesmo Kanban.
- `PATCH /api/columns/reorder`: Swap de `order_position` entre colunas.

### 3. Integridade de Dados
- **Garantia de Coluna Única:** Não será permitido deletar a última coluna de um Kanban.
- **Orquestração de Conversas:** O handler do `DELETE` na API garantirá que o campo `column_id` das conversas afetadas seja atualizado antes da remoção física do registro no banco.
