# F5.plan.006: Corrigir extractContactInfo para payload Evolution GO

**Status:** CONCLUÍDO

## Contexto

Ao receber mensagens via webhook do Evolution GO, a função `extractContactInfo()` retorna `null` porque o payload do Evolution GO usa uma estrutura completamente diferente do formato esperado (Evolution API v2 / Baileys).

**Resultado:** Contatos nunca são registrados e conversations nunca são criadas quando mensagens chegam.

**Evidência nos logs:**
```
[Webhook] MESSAGES_UPSERT: Could not extract contact info from payload
```

**Status:** 1 contact criado em 2026-04-20, conversation está parada desde então. Webhook agora funciona (200 OK), mas payload não é processado.

---

## Problema: Gap de Formato

| Campo | Formato esperado (Evolution API v2) | Formato real (Evolution GO) |
|-------|--------------------------------------|------------------------------|
| Phone/JID | `data.key.remoteJid` | `data.Info.Chat` |
| Nome | `data.pushName` | `data.Info.PushName` |
| isGroup | inferido via `@g.us` no JID | `data.Info.IsGroup` |
| fromMe | `data.key.fromMe` | `data.Info.IsFromMe` |
| Message ID | `data.key.id` | `data.Info.ID` |

### Exemplo real do payload Evolution GO:
```json
{
  "Info": {
    "Chat": "559891302872@s.whatsapp.net",
    "Sender": "559891302872@s.whatsapp.net",
    "PushName": "Renato Lhamas",
    "IsFromMe": false,
    "IsGroup": false,
    "ID": "3EB01DED7202FB093A8713",
    "Type": "text"
  },
  "Message": {
    "conversation": "teste 1010"
  },
  "IsBotInvoke": false,
  "IsDocumentWithCaption": false
}
```

---

## Solução

Modificar `extractContactInfo()` em `lib/api/webhook-utils.ts` para detectar automaticamente qual formato está presente e extrair os campos corretos.

**Lógica de detecção:**
- Se `data.Info?.Chat` existe → formato Evolution GO (novo)
- Senão se `data.key?.remoteJid` existe → formato Evolution API v2 (legado)

**Retrocompatibilidade:** A lógica permanece a mesma para o formato v2, então não quebra código existente.

---

## Arquivos a Modificar

### 1. `lib/api/webhook-utils.ts`

Atualizar função `extractContactInfo()` para suportar ambos os formatos.

**Mudança:** Adicionar detecção de `data.Info.Chat` antes da lógica de `data.key.remoteJid`.

### 2. `tests/unit/webhook-utils.test.ts`

Adicionar novo bloco `describe()` com 3 testes cobrindo o formato Evolution GO:
- Extração completa com Info.Chat + Info.PushName
- Fallback para waPhone quando PushName ausente
- Detecção de grupo via IsGroup
