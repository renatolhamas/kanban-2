# F5.plan.010 — Fix: Sanitizar NEXT_PUBLIC_APP_URL no QR route

**Data:** 2026-04-24  
**Status:** Aguardando aprovação  
**Prioridade:** 🔴 CRÍTICO

---

## Causa Raiz

`NEXT_PUBLIC_APP_URL` é lida sem `.trim()` no código. Quando a variável na Vercel tinha um `\n` (enter acidental), esse caractere foi incluído na webhook URL enviada ao EVO GO:

```
https://kanban.renatolhamas.com.br\n/api/webhooks/evo-go?tenantId=...
```

O EVO GO não consegue fazer POST para uma URL com caractere de controle inválido.

---

## Fix — 1 linha, 1 arquivo

**Arquivo:** `app/api/settings/evo-go/qr/route.ts`, linha 152

```typescript
// ANTES:
const appUrl = process.env.NEXT_PUBLIC_APP_URL;
if (!appUrl) {
  throw new Error('NEXT_PUBLIC_APP_URL is not configured');
}

// DEPOIS:
const appUrl = (process.env.NEXT_PUBLIC_APP_URL || '').trim().replace(/\/$/, '');
if (!appUrl) {
  throw new Error('NEXT_PUBLIC_APP_URL is not configured');
}
```

- Mesma variável: `NEXT_PUBLIC_APP_URL` (sem mudança nenhuma de nome)
- Continua funcionando local (`localhost:3017`) e produção (`kanban.renatolhamas.com.br`)
- `.trim()` remove qualquer `\n`, `\r` ou espaço acidental
- `.replace(/\/$/, '')` remove trailing slash

2. Escanear QR

Isso envia a URL correta ao EVO GO, sobrescrevendo o `localhost:3017` atual.
