# F5.PLAN.001 - Enhanced Webhook Registration & Routing (Stories 5.1 & 5.2)

Este plano implementa os requisitos técnicos refinados para o registro automatizado de contatos e gerenciamento de conversas via webhooks do WhatsApp (Evo GO).

## User Review Required

> [!IMPORTANT]
> **Alterações no Schema do Banco**:
> - `contacts`: Adição de `wa_name` (text) e `is_group` (boolean).
> - `conversations`: Adição de `unread_count` (int, default 0).
> **Lógica de Roteamento**: Conversas arquivadas/fechadas serão reabertas automaticamente na primeira coluna ao receberem nova mensagem. Conversas ativas permanecerão em sua posição atual.

### Database Layer (Supabase Migrations)

### Logic Layer (Utilities)

#### [MODIFY] [webhook-utils.ts](file:///c:/git/kanban.2/lib/api/webhook-utils.ts)
- **`parseJid`**: Remover sufixos de multi-dispositivo (regex: `:.*$`).
- **`extractContactInfo`**:
    - Detectar grupos via sufixo `@g.us`.
    - Exportar `waName` e booleano `isGroup`.

---

### API Layer (Webhook Handler)

#### [MODIFY] [route.ts](file:///c:/git/kanban.2/app/api/webhooks/evo-go/route.ts)
- **Tenant Lookup (4-B/A)**:
    - Prioridade: Buscar `tenants.id` via `instanceId` do payload.
    - Fallback: Parâmetro `tenantId` da URL.
- **Sincronização de Contato (Story 5.1 - Opção 1-C)**:
    - `wa_name` = `pushName` (sempre atualizado).
    - `name` = Caso `name` atual seja nulo ou registro novo, assume `pushName`. Protege edições manuais.
- **Roteamento de Conversa (Story 5.2)**:
    - Se a conversa estiver arquivada/fechada -> Move para `Main Kanban -> Primeira Coluna` e status -> `active`.
    - Se estiver ativa -> Mantém no local atual.
    - Incremento de `unread_count` para todas as novas mensagens.
