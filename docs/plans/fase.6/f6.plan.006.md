# Plano de Ação: Seletor Global de Colunas no Chat (Multi-Kanban)

**Status:** ✅ Concluído  
**Data de Conclusão:** 2026-04-30  
**Referência:** Implementado via Story 6.6.1

---

Este plano detalha as alterações necessárias para permitir que uma conversa seja movida para qualquer coluna de qualquer Kanban através do seletor no modal de chat (`ChatModal`).

## 🎯 Objetivo
Atualmente, o seletor de colunas no chat exibe apenas as colunas do Kanban atual. O objetivo é expandir essa funcionalidade para listar todos os Kanbans e suas respectivas colunas, respeitando a ordem definida nas configurações.

## 🏗️ Alterações Propostas

### 1. Novo Hook: `useKanbanStructure`
Criar um hook para buscar a hierarquia completa de Kanbans e Colunas do tenant.
- **Path:** `src/hooks/useKanbanStructure.ts`
- **Lógica:** 
  - Buscar todos os registros da tabela `kanbans` ordenados por `order_position`.
  - Incluir as colunas de cada kanban (via select aninhado ou query separada), ordenadas por `order_position`.
  - Retornar uma estrutura de dados organizada para consumo imediato pelo seletor.

### 2. Atualização da API: `update-column`
Garantir que a movimentação de uma conversa atualize tanto a coluna quanto o Kanban pai.
- **Path:** `app/api/conversations/update-column/route.ts`
- **Mudança:** Ao validar a coluna alvo, obter também o `kanban_id` associado. No comando de `update` da conversa, atualizar os campos `column_id` E `kanban_id`.
- **Racional:** Manter a integridade referencial da conversa com o board em que ela deve aparecer.

### 3. Refatoração da UI: `KanbanTransferDropdown`
Atualizar o componente de dropdown para renderizar múltiplos grupos.
- **Path:** `src/components/ui/organisms/chat/KanbanTransferDropdown.tsx`
- **Mudança:**
  - Substituir o uso de `useConversations(currentKanbanId)` pelo novo hook `useKanbanStructure`.
  - Mapear a lista de Kanbans gerando um `<optgroup>` para cada um.
  - Dentro de cada grupo, listar as colunas como `<option>`.
  - Manter o estado de loading e feedback de erro (toast) existentes.
