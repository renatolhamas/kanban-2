## Description

Implement the complete onboarding flow that creates a new tenant and its default workspace structure:

1. **Register Page** — Owner creates account with email and password
2. **Automatic Tenant Creation** — Tenant row created in `tenants` table with default name
3. **Default Kanban Setup** — "Main" kanban auto-created with standard columns (Novo, Qualificado, Em Negociação, Fechado)
4. **User Record Creation** — Owner user record created with tenant association and RLS metadata
5. **Post-Register Redirect** — User redirected to Home (dashboard) or Config page

**Important Note:**
- Story 1.1 (Database Schema with RLS) + Story 1.2 (Auth Layer) are prerequisites
- Story 1.5 ties them together at the application layer (Register UI + seed logic)
- Kanban "Main" must be marked as the default kanban for auto-routing

CREATE OR REPLACE FUNCTION create_default_kanban_on_user_insert()
RETURNS TRIGGER AS $$
BEGIN

  IF NEW.role = 'owner' THEN
    -- Create Main kanban
    INSERT INTO kanbans (tenant_id, name, is_main, order_position)
    VALUES (NEW.tenant_id, 'Main', true, 1);
    
    -- Create 4 default columns (note: need kanban_id from above)

  END IF;
  RETURN NEW;
END;
$$ LANGUAGE plpgsql;

CREATE TRIGGER trigger_create_default_kanban
AFTER INSERT ON users
FOR EACH ROW
EXECUTE FUNCTION create_default_kanban_on_user_insert();

```sql
-- Create function
CREATE OR REPLACE FUNCTION create_default_kanban_on_user_create()
RETURNS TRIGGER AS $$
DECLARE
  kanban_id UUID;
BEGIN
  -- Only create kanban for owner role (skip if user is attendant)
  IF NEW.role = 'owner' THEN
    -- Insert kanban
    INSERT INTO kanbans (tenant_id, name, is_main, order_position)
    VALUES (NEW.tenant_id, 'Main', true, 1)
    RETURNING id INTO kanban_id;
    
    -- Insert 4 default columns
    INSERT INTO columns (kanban_id, name, order_position)
    VALUES
      (kanban_id, 'Novo', 1),
      (kanban_id, 'Qualificado', 2),
      (kanban_id, 'Em Negociação', 3),
      (kanban_id, 'Fechado', 4);
  END IF;
  
  RETURN NEW;
END;
$$ LANGUAGE plpgsql;

-- Create trigger
CREATE TRIGGER trigger_create_default_kanban_on_user_insert
AFTER INSERT ON users
FOR EACH ROW
EXECUTE FUNCTION create_default_kanban_on_user_create();
```

### RLS Policies for kanbans & columns

**kanbans — Multi-tenant isolation:**
```sql
-- Users can only see kanbans for their tenant
CREATE POLICY kanban_select_by_tenant ON kanbans
  FOR SELECT
  USING (tenant_id = (auth.jwt()->>'tenant_id')::UUID);

-- Similar for INSERT, UPDATE, DELETE
```

**columns — Inherited from kanban:**
```sql
-- Users can see columns if they can see the kanban
CREATE POLICY column_select_via_kanban ON columns
  FOR SELECT
  USING (
    kanban_id IN (
      SELECT id FROM kanbans 
      WHERE tenant_id = (auth.jwt()->>'tenant_id')::UUID
    )
  );
```
