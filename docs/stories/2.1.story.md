## Description

This story establishes the **Visual DNA** of the Kanban project. It involves the technical initialization of Shadcn/ui and the translation of the "Architectural Ledger" design rules into programmatic Tailwind tokens.

**Key Technical Decisions:**
- **Initializing Shadcn:** Use `npx shadcn@latest init`.
- **Surface Hierarchy:** Implement custom Tailwind utilities for `bg-surface`, `bg-surface-low`, `bg-surface-lowest`, and `bg-surface-high`.
- **The "No-Line" Rule:** Configure the config to prioritize background shifts over border-widths.
- **Manrope Integration:** Use `next/font/google` to optimize performance and prevent FOIT.

## Dev Notes

### Color Mapping Strategy
To ensure maximum flexibility, use the HSL approach in `:root`:
```css
--primary: 161 100% 21%; /* #006c49 */
--surface: 210 20% 98%; /* #f7f9fb */
```

### Manrope Font Usage
In `tailwind.config.ts`:
```typescript
extend: {
  fontFamily: {
    sans: ["var(--font-manrope)", ...defaultTheme.fontFamily.sans],
  },
}
```
