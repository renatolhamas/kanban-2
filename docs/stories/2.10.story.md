## 🔧 Technical Approach

### Phase 1: Extend Design Tokens for Dark Mode (Tasks 1-2)

**Task 1: Add Dark Mode Tokens to Design System**

Expandir `docs/design-tokens.json`, `app/globals.css`, e `tailwind.config.ts` com variáveis de tema escuro:

1. **Color Palette (Dark Mode)**
   - Primary dark: #34D399 (Emerald 400 — otimizado para contraste em fundos escuros)
   - Secondary dark: #60A5FA (Blue 400)
   - Surface dark: #1F2937 (Gray 800 base), #111827 (Gray 900 depth) — profundidade visual
   - Text dark: #F3F4F6 (Gray 100) para body, #E5E7EB (Gray 200) para secondary
   - Semantic colors dark: Success, danger, warning ajustadas para contraste (padrão 400/500)
   - Manter paleta original intacta (light mode não muda)

2. **CSS Variables in `app/globals.css`**
   ```css
   @layer base {
     :root {
       --color-primary: #10B981;
       --color-secondary: #1E40AF;
       /* ... light mode ... */
     }
     
     [data-theme="dark"] {
       --color-primary: #34D399; /* Emerald 400 */
       --color-secondary: #60A5FA; /* Blue 400 */
       --color-surface: #111827;
       /* ... dark mode palette ... */
     }
   }
   ```

3. **Tailwind Dark Mode Configuration**
   - Usar `tailwindcss` darkMode strategy `class` (não `media`) para controle manual via UI toggle
   - Exemplo: `<html className={isDarkMode ? "dark" : ""}>`
   - Verificar `tailwind.config.ts`:
     ```javascript
     module.exports = {
       darkMode: 'class',
       theme: {
         colors: {
           primary: 'var(--color-primary)',
           dark: {
             primary: 'var(--color-primary-dark)',
             // ...
           }
         }
       }
     }
     ```

4. **Update `docs/design-tokens.json`**
   - Incluir seção `darkMode` com paleta escura completa
   - Manter estrutura consistente com light mode

**Task 2: Create Theme Toggle Component**

Implementar `<ThemeToggle />` component:

1. **Location**: `src/components/ui/ThemeToggle.tsx`
2. **Behavior**:
   - Button com ícones Sun/Moon
   - Clique alterna `data-theme="dark"` no `<html>` tag e classe `.dark` para Tailwind
   - Persiste escolha em localStorage (`theme-preference`)
   - **MANDATÓRIO**: No mount, detecta preferência do sistema (`window.matchMedia('(prefers-color-scheme: dark)')`) se localStorage estiver vazio
3. **Accessibility**:
   - ARIA label: "Mudar para tema escuro" / "Mudar para tema claro"
   - Keyboard accessible (Tab + Enter/Space)
4. **Integration**:
   - Integrar no **Header** (lado direito, próximo ao perfil/busca) para máximo visibility

### Phase 2: Update Components & Storybook (Tasks 3-4)

**Task 3: Update All Component Stories for Dark Mode**

Para cada arquivo `.stories.tsx`:

1. **Add Dark Mode Decorator** (Storybook best practice)
   ```typescript
   export default {
     title: 'Components/Button',
     decorators: [
       (Story) => (
         <div className="space-y-4">
           <div>
             <h3 className="text-sm font-semibold mb-2">Light Mode</h3>
             <Story />
           </div>
           <div className="dark bg-slate-900 p-4 rounded">
             <h3 className="text-sm font-semibold text-white mb-2">Dark Mode</h3>
             <Story />
           </div>
         </div>
       ),
     ],
   }
   ```

2. **Validate Component Rendering**
   - Componentes com `className` dark-aware já funcionam (via Tailwind prefix `dark:`)
   - Exemplo: `<button className="bg-white dark:bg-slate-900 text-black dark:text-white">`
   - Verificar ausência de hardcoded colors em inline styles

3. **Test All Variants**
   - Cada story (Primary, Secondary, Danger, etc.) renderiza em light e dark
   - Hover/focus states funcionam em ambos os temas
   - Disabled states visíveis em ambos

**Task 4: Update DesignTokens.stories.tsx with Dark Mode**

Estender a story criada em 2.9:

1. **Color Swatches Section (Dark Mode)**
   - Adicionar coluna comparativa: Light vs Dark swatches lado-a-lado
   - Mostrar contrast ratios em AMBOS os contextos
   - Exemplo:
     ```
     Primary (Emerald):
     - Light: #10B981 on #F9FAFB (contrast 7.2:1 ✅)
     - Dark:  #34D399 on #111827 (contrast 5.1:1 ✅)
     ```

2. **Typography in Dark Mode**
   - Exibir fonts com fundo escuro
   - Validar legibilidade em ambos

3. **Spacing Grid in Dark Mode**
   - Boxes com background escuro mostrando espaçamento

4. **Theme Toggle Inline**
   - Botão para alternar light/dark na própria story
   - Mostra mudança em tempo real

### Phase 3: Accessibility Validation (Tasks 5-6)

**Task 5: WCAG AA Audit for Dark Mode**

1. **Contrast Ratio Validation**
   - Rodar `axe-core` ou `@axe-core/react` — para accessibility auditing
   - `color-contrast` — para validação rápida de contrast ratios
   - `chroma.js` ⭐ **SELECTED** — biblioteca oficial para manipulação de cores e simulação de daltonismo
     - **Motivo**: Precisão matemática superior para espaços de cores e CVD (Color Vision Deficiency) simulation.
     - **Ref**: [chroma.js docs]
   - Validar TODAS as combinações foreground/background
   - Resultado esperado: Todas >= 4.5:1 (AA), idealmente >= 7:1 (AAA)
   - Documentar em tabela: `docs/qa/dark-mode-contrast-matrix.md`

2. **AXE DevTools Scan**
   ```bash
   npm run test:a11y -- --tags wcag2aa
   # OR manual: navegar Storybook stories e rodar AXE extension
   ```
   - Esperado: 0 violations críticas
   - Regiões/landmarks corretas
   - Labels de formulário presentes

3. **WAVE Evaluation**
   - Navegar `http://localhost:6006` (Storybook)
   - Rodar WAVE browser extension
   - Verificar: contrast, missing alternatives, structural

**Task 6: Keyboard Navigation & ARIA Comprehensive Test**

1. **Keyboard Navigation Matrix**
   - Teste cada componente: Button, Input, Tabs, Modal, etc.
   - Tab order correto (left-to-right, top-to-bottom)
   - Focus visible em light E dark mode
   - Esc fecha modais/dropdowns
   - Enter submete forms
   - Arrow keys funcionam em tabs/menus
   - Documento: `docs/qa/keyboard-navigation-matrix.md`

2. **ARIA Verification**
   - Todos os componentes têm `role` apropriado (button, navigation, tablist, etc.)
   - Inputs têm `<label>` ou `aria-label`
   - Modais têm `role="dialog"` + `aria-labelledby` + `aria-describedby`
   - Mensagens de erro associadas via `aria-describedby`
   - Lists/tables têm `<ul>`/`<table>` semânticos

### Phase 4: Documentation & Finalization (Task 7)

**Task 7: Documentation Update**

1. **Update `docs/DESIGN-TOKENS.md`**
   - Adicionar seção "Dark Mode" com:
     - Screenshots de dark mode theme toggle
     - Paleta completa (light + dark lado-a-lado)
     - Guia: "Como aplicar dark mode em novos componentes"
     - Exemplo: `className="bg-white dark:bg-slate-900 text-black dark:text-white"`

2. **Accessibility Checklist in `DESIGN-TOKENS.md`**
   - Consolidar findings de 2.9 + 2.10:
     - ✅ WCAG AA contrast (4.5:1 mínimo)
     - ✅ 0 critical AXE violations
     - ✅ Keyboard accessible
     - ✅ Semantic HTML
     - ✅ ARIA labels present
   - Tabela de conformidade por componente

3. **Dark Mode Implementation Guide**
   - Steps para aplicar dark mode em novo componente:
     1. Use Tailwind dark prefix: `dark:bg-slate-900`
     2. Test em Storybook com dark decorator
     3. Verificar contraste via AXE
   - Exemplos de padrões comuns (backgrounds, text, borders)
