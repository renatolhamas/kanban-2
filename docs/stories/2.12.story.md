## 🔧 Technical Approach

### Overall Strategy: 7-Page Systematic Refactor

**Canonical Source of Truth:** `src/docs/` (Design System)
- 8 Atoms: Button, Input, Card, Textarea, Badge, Skeleton, Spinner, ThemeToggle
- Import path: `@/components/ui/{component}`
- Dark mode: CSS variables + `dark:` Tailwind prefixes
- Accessibility: WCAG AA via aria-describedby, semantic HTML, keyboard nav

**Refactor Pattern (Apply to ALL 7 pages):**
1. Remove legacy component imports (PasswordInput, FormError, custom components)
2. Import ONLY 8 Atoms from `@/components/ui/`
3. Replace hardcoded colors with Tailwind dark: prefixes
4. Implement semantic form labels + aria-describedby for inline errors
5. Remove NO-OP Toast calls (not part of 8 Atoms)
6. Verify keyboard navigation, dark mode, contrast

- `app/(auth)/login/page.tsx`
- `app/(auth)/register/page.tsx`

- `app/(auth)/forgot-password/page.tsx`
- `app/(auth)/resend-confirmation/page.tsx`

**Task 2.1-2.5: Refactor Each Auth Page**

For EACH page:
1. Audit imports: Remove PasswordInput, FormError, custom components
2. Use `@/components/ui/input`, `@/components/ui/button`, `@/components/ui/card`
3. Replace inline errors with `aria-describedby` pattern
4. Remove Toast/useToast
5. Add `dark:` Tailwind prefixes
6. Verify keyboard navigation (Tab through all fields)

**Example Pattern:**
```tsx
// OLD
import { PasswordInput } from '@/components/PasswordInput'
import { FormError } from '@/components/FormError'

// NEW
import { Input } from '@/components/ui/input'
import { Button } from '@/components/ui/button'

// Error handling
<Input
  id="password"
  type="password"
  aria-describedby={passwordError ? 'password-error' : undefined}
/>
{passwordError && (
  <span id="password-error" role="alert" className="text-red-500 text-sm">
    {passwordError}
  </span>
)}
```
