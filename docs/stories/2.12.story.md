# Story 2.12 — Pages Refactor — Application Pages with Design System

## 📖 Story

**As a** Designer/Frontend Team,  
**I want** all application pages to use consistent Design System components,  
**so that** the UI is visually consistent, accessible, and maintainable across light/dark modes

---

## 📓 Decision Log (Atlas Analyst)

| Item | decision | Rationale |
|------|----------|-----------|
| **Profile Layout** | RootLayout Integration | Profile is part of the logged-in experience; Header/Sidebar provide consistent navigation. |
| **Source of Truth** | `src/components/ui/` | Standardizing on the shadcn/ui folder structure for all core components. |
| **Home/Landing** | Isolated Marketing Page | Root `/` serves as a clean landing page for unauthenticated users, without sidebar. |
| **Error Feedback** | Dual (Inline + Toast) | Follows the established pattern: Inline for field validation, Toast for API/Global messages. |
| **Cleanup** | Immediate Deletion | Deprecated components (PasswordInput, FormError) are removed to maintain a clean codebase. |

## 📋 Overview

Refatorar as páginas existentes da aplicação que ainda usam componentes e estilos antigos, integrando completamente os componentes validados do Design System (Stories 2.1-2.9) e o novo Application Layout estabelecido em Story 2.11.

**Páginas a refatorar (fora do padrão do Design System):**
1. `app/page.tsx` — Landing/Home page (usa Tailwind inline, sem componentes Design System)
2. `app/profile/page.tsx` — User profile page (usa PasswordInput customizado antigo, FormError, sem Design System)

**Objetivo:** Garantir que toda a interface de usuário seja visualmente consistente com o Design System, acessível em dark/light mode, e pronta para a integração de dados das próximas epics.

## 🔧 Technical Approach

### Phase 1: Refactor `app/page.tsx` (Landing/Home Page)

**Current State**: Uses inline Tailwind classes (bg-blue-600, px-6, py-3) com hardcoded colors

**Target State**: Use Button component do Design System

**Task 1.1: Refactor Home Page**

1. **File**: `app/page.tsx`
2. **Changes Required**:
   - Replace hardcoded link buttons with `Button` component from Design System
   - Import: `import { Button } from '@/src/components/ui/button'` (padrão shadcn)
   - Replace:
     ```tsx
     // OLD
     <a className="px-6 py-3 bg-blue-600 text-white rounded-lg hover:bg-blue-700">
     
     // NEW
     <Button variant="primary" asChild>
       <a href="/register">Register</a>
     </Button>
     ```
   - Use Tailwind design tokens (`dark:` prefixes) para dark mode suporte
   - Remover classes hardcoded de cor

3. **Validation**:
   - Render em Storybook
   - Testar light/dark mode
   - Verificar button states (hover, active, disabled)

### Phase 2: Refactor `app/profile/page.tsx` (User Profile Page)

**Current State**: 
- Usa `PasswordInput` customizado (antigo componente em `components/PasswordInput.tsx`)
- Usa `FormError` customizado (em `components/FormError.tsx`)
- Hardcoded colors: `bg-gray-50`, `bg-blue-600`, `text-gray-700`, `border-red-500`
- Sem dark mode support
- Sem RootLayout (sem Header/Sidebar)

**Target State**: 
- Use `Input` component do Design System
- Use `Toast` component para error/success messages
- Usar design tokens para cores
- Integrar com RootLayout (Story 2.11)

**Task 2.1: Refactor Profile Page Structure**

1. **File**: `app/profile/page.tsx`
2. **Changes Required**:

   a) **Import Design System components**:
   ```tsx
   import { Input } from '@/src/components/ui/input'    // Replace PasswordInput
   import { Button } from '@/src/components/ui/button'
   import { Card } from '@/src/components/ui/card'
   import { useToast } from '@/src/components/ui/use-toast' // For API/Global feedback
   ```

   b) **Replace PasswordInput sections**:
   ```tsx
   // OLD
   <PasswordInput
     id="new-password"
     name="new-password"
     value={newPassword}
     onChange={setNewPassword}
     label="New Password"
     showStrength={true}
     showRequirements={true}
   />
   
   // NEW
   <Input
     id="new-password"
     type="password"
     value={newPassword}
     onChange={(e) => setNewPassword(e.target.value)}
     label="New Password"
     placeholder="Enter new password"
     error={passwordError} // Inline error support
   />
   ```

   c) **Implement Dual Feedback (Toast + Inline)**:
   ```tsx
   // Success/API Error via Toast
   const { toast } = useToast()
   // toast({ title: 'Success', description: 'Profile updated' })

   // Validation errors via Input error prop
   <Input error={emailError} ... />
   ```

   d) **Remove hardcoded colors**, use Tailwind dark mode:
   ```tsx
   // OLD
   className="min-h-screen bg-gray-50"
   
   // NEW
   className="min-h-screen bg-gray-50 dark:bg-gray-900"
   ```

   e) **Integrate RootLayout**:
   ```tsx
   // Envolver conteúdo com RootLayout para Sidebar/Header consistentes
   import RootLayout from '@/src/components/layout/RootLayout'
   ```

3. **Validation**:
   - Form still functions (save/cancel)
   - Errors display via Toast
   - Light/dark mode rendering correct
   - Contrast >= 4.5:1 in both themes
