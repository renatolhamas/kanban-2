## Dev Notes

### Technical Context (from Architecture Docs)
- **Tech Stack:** Next.js 14, Vitest, React Testing Library. [Source: docs/architecture/1-tech-stack-strategic-decisions.md]
- **Project Structure:** Tests should reside alongside components or in the `tests/` directory as appropriate. Component tests use `.test.tsx`. [Source: docs/architecture/2-project-structure-source-tree.md]
- **Testing Goal:** High-speed feedback loop for component logic before browser-based verification.

### Testing Strategy
- Use **jsdom** as the primary environment for component logic (Industry standard compliance).
- Use **Storybook Portable Stories** (@storybook/addon-vitest) for verifying components in the real browser environment (Playwright) as configured in Story 2.2.
- This story bridges the gap by adding "Fast Unit Tests" to the already existing "Reliable Browser Tests".

### File Locations
- **Configuration:** `vitest.config.ts`
- **Setup:** `tests/setup.ts` (NEW)
- **Tests:** `components/common/AtmosphereCheck.test.tsx` (NEW)
