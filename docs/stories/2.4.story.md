## Story

**As a** developer,
**I want** to create core form atom components (Button, Input, Toast) with Shadcn/ui following the "Architectural Ledger" design system,
**so that** I have reusable, accessible, and visually consistent form building blocks for the authentication refactor and dashboard features.

## Acceptance Criteria

### 🎨 Component Library — Button
- [ ] `Button.tsx` component created using Shadcn/ui base with these variants:
  - `variant="primary"` (Emerald #10b981, SemiBold text, no borders)
  - `variant="secondary"` (Navy #515f78, high contrast for actions)
  - `variant="ghost"` (transparent, underline on hover)
  - `variant="disabled"` (greyed out, no interactions)
- [ ] All variants respect the 8px border radius rule
- [ ] **No 1px borders** — use tonal surface shifts from the Hierarchy
- [ ] **Hover Interactions:**
  - `variant="primary"`: Color shift to Deep Emerald (#006c49)
  - `variant="secondary"`: Color shift to Deep Navy (#434f63)
  - `variant="ghost"`: Surface shift to `bg-surface-high` (#e6e8ea)
  - Hover states should use subtle transitions (150ms)

### 🎨 Component Library — Input
- [ ] `Input.tsx` component created with:
  - Default state: `bg-surface-lowest` (#ffffff) with 2px bottom indicator using `bg-surface-high` (#e6e8ea) — avoid 1px lines
  - Focus state: Emerald halo ring (`ring-2 ring-emerald-500/20`) and subtle elevation shadow (no thick borders)
  - Error state: Validation feedback with red/orange indicator (color-accessible)
  - Label association and `aria-label` support
  - Placeholder text and helper text slots
- [ ] Input respects Manrope typography (base 16px)

### 🎨 Component Library — Toast
- [ ] `Toast.tsx` context/hook system created for notifications:
  - Variants: `success`, `error`, `warning`, `info`
  - Auto-dismiss after 5 seconds (configurable)
  - Close button on each toast
  - Stack multiple toasts on **Top-Right** of the screen (max 3 visible)
- [ ] **Glassmorphism Aesthetic:** Toast uses `bg-surface/80` with `backdrop-blur-md` (no outlines)
- [ ] Accessibility: `role="alert"` and `aria-live="polite"`

### 📖 Storybook Integration
- [ ] Storybook stories created for each component:
  - Button: all 4 variants, disabled state, size variants (sm, md, lg)
  - Input: default, focus, error, disabled, with label, with helper text
  - Toast: all 4 types, multiple stacks, auto-dismiss demo
- [ ] Each story demonstrates the component in the Design System context
- [ ] Stories viewable in Storybook (from Story 2.2 setup)

### ✅ Testing & Quality
- [ ] Unit tests created for each component:
  - Button: click handlers, variant rendering, disabled state
  - Input: value changes, placeholder, error state, label binding
  - Toast: create/dismiss, auto-close timing, stacking
- [ ] All tests pass with `npm run test`
- [ ] Components render without errors in Vitest jsdom environment (from Story 2.3)
- [ ] Accessibility checks pass: color contrast WCAG AA, ARIA attributes present

### 📋 Documentation
- [ ] Component file locations documented in story
- [ ] Design Token usage noted in each component
- [ ] Example usage snippets in component comments

## Dev Notes

### Technical Context (from Architecture Docs)
- **Design System:** Shadcn/ui components styled with Tailwind CSS + Design Tokens from Story 2.1
- **Palette:** Emerald (#10b981 action, #006c49 primary), Navy (#515f78 secondary)
- **Typography:** Manrope font, SemiBold for actions
- **Surface Hierarchy:** bg-surface, bg-surface-low, bg-surface-lowest, bg-surface-high
- **Testing:** Vitest + React Testing Library (Story 2.3), Storybook Portable Stories (Story 2.2)
- **Spacing:** Editorial scale: 2.5rem (10) to 4rem (16) for sections

### Design Rules (from Epic 2)
- **No 1px Borders Rule:** Visual definition comes from tonal surface shifts, 2px indicators, and color changes, not lines
- **8px Radius Rule:** All components use `rounded-lg` (8px) or larger
- **Glassmorphism Ready:** Toasts use `bg-surface/80` + `backdrop-blur-md` as the system standard for feedback layers

### File Locations
- **Components:** `app/components/common/Button.tsx`, `Input.tsx`, `Toast.tsx`
- **Hooks:** `app/hooks/useToast.ts`
- **Stories:** `app/components/common/*.stories.tsx`
- **Tests:** `app/components/common/*.test.tsx`, `app/hooks/*.test.tsx`
