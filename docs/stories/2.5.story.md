## Dev Notes

### Technical Context (from Architecture Docs)
- **Auth:** Supabase Cloud Auth with JWT tokens
- **API:** Next.js API Routes for auth endpoints
- **Design System:** Shadcn/ui + Tailwind CSS (Story 2.1-2.4)
- **Components:** Button, Input, Toast from Story 2.4 (ready to use)
- **Testing:** Vitest + React Testing Library (Story 2.3), Storybook (Story 2.2)

### Design Rules (from Epic 2)
- **No 1px Borders Rule:** Use tonal surface shifts and 2px indicators instead
- **8px Radius Rule:** All components use `rounded-lg` or larger
- **Glassmorphism Toast:** `bg-surface/80` + `backdrop-blur-md` for feedback layers
- **Button State Management:** Must re-enable on error; stay disabled on success until redirect
- **Navigation Links:** Ghost button variant for "Sign up" handles should include a trailing icon (e.g., `ArrowRight`)
- **Manrope Typography:** SemiBold for headings, Regular for body text
- **Editorial Spacing:** 2.5rem-4rem for section margins

### Component Reuse
- **Button.tsx:** primary (submit), secondary (alternative), ghost (links)
- **Input.tsx:** email, password fields with validation states
- **Toast.tsx:** success/error/warning notifications with auto-dismiss
- **Spinner Implementation:** Use Lucide-react `Loader2` icon with Tailwind `animate-spin` class
  - Example: `<Loader2 className="animate-spin" />`
  - No new Spinner component needed — reuse from Story 2.4 dependencies
  - Icon color should match button text color (inherit or explicit)

### File Locations
- **Pages:** `app/(auth)/register/page.tsx`, `app/(auth)/login/page.tsx`
- **Stories:** `app/(auth)/*.stories.tsx`
- **Tests:** `app/(auth)/*.test.tsx`
