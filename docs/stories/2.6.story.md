## Acceptance Criteria

### 🎨 Card Component (Conversation & Info containers)
- [ ] Implement `Card` component using tonal layering:
  - `bg-surface-lowest` (#ffffff) when placed on a `bg-surface` (#f7f9fb) background.
  - **No 1px borders** (strictly prohibited).
  - 8px radius (`rounded-lg`).
- [ ] Sectioning inside Card:
  - Separate header from body using a **1.5rem (6) vertical spacing gap** (Editorial constraint).
  - Use tonal shifts to `bg-surface-low` strictly for functional footers or distinct action bars.
- [ ] Support for high-priority variant: 4px Emerald left-accent bar ("Executive Insight" pattern).

### 🎨 Modal Component (Dialog System)
- [ ] Implement `Modal` (Dialog) component following the glassmorphism rule:
  - Backdrop: Blurred overlay with tonal transparency.
  - Container: 8px radius, `bg-surface-lowest`.
  - Elevation: Soft-breath shadow `0px 12px 32px rgba(10, 25, 47, 0.06)`.
- [ ] Sticky Header: **Must implement** a fixed header with glass property (80% opacity + 20px backdrop blur).
- [ ] Animation: **Combined Fade + Scale** (starting at 95% scale) with 200ms duration for a premium entrance effect.

### 🎨 Tabs Component (Interface Navigation)
- [ ] Implement `Tabs` for section navigation (Profile, Connection, etc.):
  - Active state: Subtle background tonal shift + Manrope SemiBold typography.
  - Inactive state: Regular typography, `text-secondary`.
  - **No bottom-border logic**; use surface contrast to denote selection.

### 🎨 Avatar & Badge (Identity & Status)
- [ ] Implement `Avatar` component:
  - Support for src image and name-based textual fallback logic.
  - **Primary Variant:** `rounded-lg` (8px) following project radius rule.
  - **Secondary Variant:** Circle (only if explicitly requested in sub-components).
- [ ] Implement `Badge` component for metadata:
  - 9999px (full) roundedness.
  - Tonal variants: Emerald (Positive), Steel/Navy (Neutral), Amber (Warning), Red (Error).
  - Use `surface-variant` backgrounds with high-contrast text.

### ♿ Accessibility & Quality
- [ ] Color contrast passes WCAG AA for all variants.
- [ ] Interactive elements (Tabs, Modal) support keyboard navigation (Tab, Enter, Esc).
- [ ] No hardcoded colors — all use Design Token CSS variables.

## Dev Notes

### Technical Context (from Architecture Docs)
- **Framework:** Next.js + Tailwind CSS + Shadcn/ui.
- **Typography:** Manrope (Display: ExtraBold, Headline: SemiBold, Body: Regular/Medium).
- **Design Rules:**
  - **No-Line Rule:** Boundaries defined by `bg-surface` hierarchy shifts. [Source: docs/DESIGN-PRINCIPLES.md#2-cores-e-a-regra-do-no-line]
  - **8px Radius Rule:** All components use `rounded-lg`. [Source: docs/DESIGN-PRINCIPLES.md#5-componentes-padrões-executivos]
  - **Surface Hierarchy:** Base (#f7f9fb), Low (#f2f4f6), Lowest (#ffffff), High (#e6e8ea).
- **File Locations:** Core components in `components/common/`. Stories/Tests alongside components or in `components/common/*.test.tsx`.

### Testing Standards
- **Framework:** Vitest + React Testing Library.
- **Standards:** All components must have a basic "smoke test" and a "behavioral test" (interaction checks).
- **Accessibility:** `aria-*` attributes required for Modal and Tabs.

### Dependency Chain
- **Requires:** Story 2.1-2.4 (Foundational setup & Atomic components) ✅
- **Blocks:** Story 2.9 (App Layout), Epic 4 (Kanban Board)
