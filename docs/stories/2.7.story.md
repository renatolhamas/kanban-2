## Story

**As a** designer and developer,
**I want** to establish a comprehensive token system with 30+ design tokens (spacing, typography, shadows, colors, animation) and export them in W3C DTCG format,
**so that** the Design System has a scalable, programmatic foundation that enables consistent application across all components and supports future design tool integrations.

### ⚙️ Expand `tailwind.config.ts` with Token Utilities

- [ ] Create Tailwind theme config section extending default colors, spacing, fontSize, lineHeight
- [ ] Map all CSS variable tokens to Tailwind utilities (e.g., `spacing: { ... }`, `colors: { ... }`, `animation: { ... }`)
- [ ] Ensure `npm run build` passes with 0 errors and 0 warnings
- [ ] Verify Tailwind utilities reference correct CSS variables without hardcoded values

### 📋 Create W3C DTCG Export Format

- [ ] Create `docs/design-tokens.json` in W3C Design Tokens Community Group (DTCG) format
- [ ] Include all 40+ tokens with metadata:
  - `$type` (e.g., "dimension", "color", "typography", "shadow", "animation")
  - `$value` (actual token value, e.g., "0.5rem", "#10B981", "0 12px 32px rgba(...)")
  - `$description` (when/why to use the token)
- [ ] Example structure:
  ```json
  {
    "spacing": {
      "0": { "$value": "0", "$type": "dimension", "$description": "No spacing" },
      "1": { "$value": "0.25rem", "$type": "dimension", "$description": "Extra small spacing (4px)" },
      ...
    },
    "color": {
      "primary": { "$value": "#10B981", "$type": "color", "$description": "Primary action color (Emerald 500)" },
      ...
    }
  }
  ```

### 📦 Create Token Export Scripts

- [ ] Create `scripts/export-tokens.js` that:

  - Generates `exports/tokens.css` (CSS variables for browser consumption)

  - Generates `exports/tokens.d.ts` (TypeScript type definitions)
- [ ] Ensure all exports are valid and aligned with source tokens

## Dev Notes

### Architecture References

- **Tech Stack (Design System):** [Source: architecture/1-tech-stack-strategic-decisions.md#12-color-palette--typography]
  - Tailwind CSS + shadcn/ui foundation
  - Architectural Ledger colors: Primary (#10B981), Secondary (#1E40AF), Danger (#EF4444), Success (#22C55E)
  - Typography: Manrope font family, font sizes 12px → 32px, line heights 1.5 (body) / 1.2 (headings)

- **Component Pattern References:**
  - Story 2.6 Modal shadows: `0px 12px 32px rgba(10, 25, 47, 0.06)` — reference for shadow tokens
  - Card component uses 8px radius and tonal layering (see Story 2.6 AC) — spacing tokens should support this

- **W3C DTCG Format:**
  - Industry-standard format for design tool integration (Figma, Adobe XD, etc.)
  - Each token includes `$type`, `$value`, `$description` metadata
  - Enables future token sync workflows and design-to-code tooling

### Previous Story Insights

- Story 2.6 (UI Expansion) validated Tailwind + shadcn/ui setup
- Components created in 2.1-2.6 used inline spacing/colors but relied on utility classes
- Token system will provide explicit, reusable design values for audit and documentation (Stories 2.8-2.9)

### File Structure

- **Token Definition:** `docs/design-tokens.json` (W3C DTCG format)
- **CSS Export:** `exports/tokens.css` (CSS custom properties)
- **JavaScript Export:** `exports/tokens.js` (JS object)
- **TypeScript Definitions:** `exports/tokens.d.ts` (TS types)
- **Export Script:** `scripts/export-tokens.js` (Node.js automation)

### Testing Requirements

- CSS variable validation: Ensure all tokens in `globals.css` follow `--category-token-name` pattern
- Tailwind build: No errors or warnings during build
- Export script: Validates JSON format and generates all three file outputs
- TypeScript: All token types properly exported
