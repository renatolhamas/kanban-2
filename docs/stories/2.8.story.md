## 🔧 Technical Approach

### Phase 1: Component Audit (Task 1)

**Objetivo:** Mapear uso de espaçamento, cores e tipografia em componentes 2.1-2.6

**Steps:**
1. Listar todos os componentes criados em Stories 2.1-2.6:
   - Button, Input, Toast (2.4)
   - Card, Modal, Tabs, Avatar, Badge (2.6)
   - Auth Form components (2.5)
2. Para cada componente:
   - Inspecionar espaçamento (padding, margin, gap)
   - Validar contra scale: 8px, 16px, 24px, 32px, 48px, 64px
   - Registrar deviações (ex: 12px, 20px)
   - Justificar se intencional ou corrigir
3. Auditar cores:
   - Comparar com tokens definidos em `tailwind.config.ts`
   - Validar contrast ratios (WCAG AA: 4.5:1 text, 3:1 graphics)
4. Auditar tipografia:
   - Verificar font sizes, weights, line heights contra tokens
   - Validar legibilidade em mobile (18px mínimo)

### Phase 2: Documentation (Tasks 2-5)

**Task 2: Create DESIGN-TOKENS.md**
- Estrutura markdown com seções:
  1. **Color System** (primary, secondary, neutral, semantic)
  2. **Typography** (scales, font families, weights)
  3. **Spacing** (8px scale visual)
  4. **Shadows** (elevation levels)
  5. **Animation** (timing, easing)
- Incluir:
  - Hex codes, RGB, CSS variable names
  - Use cases ("quando usar cada cor")
  - Examples (componentes que usam cada token)
  - Contrast ratios com PASS/FAIL badges

**Task 3: Export & Validate Tokens**
- Confirmar `docs/design-tokens.json` tem 100% dos tokens com `$description`
- Rodar `npm run tokens:export` e validar outputs:
  - `exports/tokens.css` (CSS variables)
  - `exports/tokens.js` (JS object)
  - `exports/tokens.d.ts` (TypeScript types)
- Validar Tailwind ainda consegue resolver todos os tokens

**Task 4: Spacing Consistency Audit**
- Mapear uso de espaçamento em cada componente
- Criar tabela:
  | Component | Padding | Margin | Gap | Deviations | Justified? |
  |-----------|---------|--------|-----|-----------|-----------|
- Desvios documentados e justificados (ou corrigidos se erro)

**Task 5: Create Spacing Visual Reference**
- Gerar imagem ou CSS visual mostrando:
  - 8px scale (8, 16, 24, 32, 48, 64px boxes)
  - Comparação visual com componentes
  - Rótulos com CSS variable names
- Incluir em `docs/DESIGN-TOKENS.md` para fácil referência

**Task 6: Create COMPONENT-TOKEN-CHECKLIST.md**
- Checklist reutilizável para novos componentes:
  - [ ] Espaçamento usa escala 8px
  - [ ] Cores com contrast ratio ≥4.5:1
  - [ ] Tipografia segue escala definida
  - [ ] Sombras usam elevation tokens
  - [ ] Animações usam duração/easing padrão
  - [ ] Testado em dark mode (se aplicável)
