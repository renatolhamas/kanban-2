## Technical Architecture

### Endpoint Details (from Story 3.2)

**Method:** POST  
**URL:** `/api/webhooks/evo-go?tenantId={tenantId}`  
**Body:**
```json
{
  "event": "CONNECTION_UPDATE",
  "data": {
    "statusConnection": "connected",
    "instance": "instance-123"
  }
}
```

**Processing Flow:**
1. Validate HMAC-SHA256 signature (X-Signature header)
2. Extract tenantId from query parameter
3. Update `tenants.connection_status` based on event type
4. Return 200 OK

### Security

- **HMAC-SHA256:** Signature validation prevents spoofing (implemented in Story 3.2 + 3.1)
- **Tenant Isolation:** WHERE tenantId enforced on all mutations
- **Error Messages:** Generic responses (no internal details leaked)
