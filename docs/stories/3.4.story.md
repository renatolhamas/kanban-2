## Technical Architecture

### Signature Validation Flow

```
POST /api/webhooks/evo-go
  ↓
Extract X-Signature header
  ↓
Validate signature using HMAC-SHA256
  ├─ If invalid: return 401 Unauthorized
  └─ If valid: proceed to event processing
  ↓
Process CONNECTION_UPDATE event
  ↓
Update DB + return 200 OK
```

### Security Properties

- **Algorithm:** HMAC-SHA256 (cryptographically secure)
- **Constant-Time:** Uses `crypto.timingSafeEqual()` to prevent timing attacks
- **Secret Management:** Stored in `EVO_GO_WEBHOOK_SECRET` environment variable
- **No Logging:** Signatures never logged (security best practice)

### Why Constant-Time Comparison?

Timing attacks exploit the fact that naive string comparison (`===`) fails fast when characters differ. By comparing fixed-length HMAC results in constant time:
- Attacker cannot determine which character is wrong
- Prevents side-channel attacks via response time analysis
