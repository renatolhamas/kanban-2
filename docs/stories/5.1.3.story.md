# Story 5.1.3: Corrigir extractContactInfo para payload Evolution GO

> [!IMPORTANT]
> Esta story resolve um gap crítico: a função `extractContactInfo()` retorna `null` ao receber mensagens via webhook do Evolution GO porque o payload usa uma estrutura completamente diferente do formato esperado (Evolution API v2 / Baileys). **Resultado:** Contatos nunca são registrados e conversations nunca são criadas quando mensagens chegam via Evo GO.

## Executor Assignment

executor: "@dev"
quality_gate: "@architect"

## Story Statement

**As a** system,
**I want to** automatically detect and correctly extract contact information from both Evolution API v2 and Evolution GO webhook payloads,
**so that** contacts are properly registered and conversations are created regardless of which Evolution platform sends the message.

---

## Scope

**IN:**
- Detectar automaticamente o formato do payload (Evolution GO vs Evolution API v2)
- Extrair `phone/JID` de `data.Info.Chat` (Evo GO) ou `data.key.remoteJid` (v2)
- Extrair `nome` de `data.Info.PushName` (Evo GO) ou `data.pushName` (v2)
- Detectar grupo via `data.Info.IsGroup` (Evo GO) ou inferir de `@g.us` no JID (v2)
- Extrair `isFromMe` de `data.Info.IsFromMe` (Evo GO) ou `data.key.fromMe` (v2)
- Extrair `messageId` de `data.Info.ID` (Evo GO) ou `data.key.id` (v2)
- Manter retrocompatibilidade com formato v2 (nenhuma quebra de código existente)
- Adicionar testes cobrindo ambos os formatos

**OUT:**
- Mudanças em webhook contracts público
- Refactor completo de webhook handler
- Suporte a outros formatos Evolution além v2 e GO

## Dev Notes

### F5.plan.006 — Gap de Formato extractContactInfo

**Problema:**
- Webhook Evo GO chega com estrutura diferente: `data.Info.Chat`, `data.Info.PushName`, etc
- Código atual espera: `data.key.remoteJid`, `data.pushName` (Evolution API v2 / Baileys)
- Resultado: `extractContactInfo()` retorna `null`, contatos não são criados, webhook falha silenciosamente

**Evidência nos logs:**
```
[Webhook] MESSAGES_UPSERT: Could not extract contact info from payload
```

**Solução:**
Modificar `extractContactInfo()` em `lib/api/webhook-utils.ts` para detectar automaticamente qual formato está presente e extrair os campos corretos.

**Lógica de detecção:**
```
SE data.Info?.Chat existe:
  → Formato Evolution GO (novo)
  → Extrai campos de data.Info.*
SENÃO SE data.key?.remoteJid existe:
  → Formato Evolution API v2 (legado)
  → Extrai campos de data.key.*, data.pushName, etc
SENÃO:
  → Formato desconhecido
  → Lança erro detalhado
```

**Mapeamento de Campos:**

| Campo Lógico | Evolution API v2 | Evolution GO | Evo GO Fallback |
|---|---|---|---|
| Phone/JID | `data.key.remoteJid` | `data.Info.Chat` | — |
| Nome | `data.pushName` | `data.Info.PushName` | Extrair de JID como `waPhone` |
| IsGroup | Inferir de `@g.us` no JID | `data.Info.IsGroup` | — |
| IsFromMe | `data.key.fromMe` | `data.Info.IsFromMe` | — |
| Message ID | `data.key.id` | `data.Info.ID` | — |

**Exemplo real do payload Evolution GO:**
```json
{
  "Info": {
    "Chat": "559891302872@s.whatsapp.net",
    "Sender": "559891302872@s.whatsapp.net",
    "PushName": "Renato Lhamas",
    "IsFromMe": false,
    "IsGroup": false,
    "ID": "3EB01DED7202FB093A8713",
    "Type": "text"
  },
  "Message": {
    "conversation": "teste 1010"
  },
  "IsBotInvoke": false,
  "IsDocumentWithCaption": false
}
```

**Implementação esperada:**
```typescript
export interface ContactInfo {
  phone: string;
  pushName: string;
  isGroup: boolean;
  isFromMe: boolean;
  messageId: string;
}

export function extractContactInfo(data: any): ContactInfo {
  // Detect format
  const isEvoGo = data?.Info?.Chat != null;
  
  if (isEvoGo) {
    // Evolution GO format
    const phone = data.Info.Chat;
    const pushName = data.Info.PushName || extractPhoneFromJid(phone);
    const isGroup = data.Info.IsGroup === true;
    const isFromMe = data.Info.IsFromMe === true;
    const messageId = data.Info.ID;
    
    return { phone, pushName, isGroup, isFromMe, messageId };
  } else if (data?.key?.remoteJid != null) {
    // Evolution API v2 format (legacy)
    const phone = data.key.remoteJid;
    const pushName = data.pushName || extractPhoneFromJid(phone);
    const isGroup = phone.includes('@g.us');
    const isFromMe = data.key.fromMe === true;
    const messageId = data.key.id;
    
    return { phone, pushName, isGroup, isFromMe, messageId };
  } else {
    throw new Error('Unable to extract contact info: unknown payload format');
  }
}

function extractPhoneFromJid(jid: string): string {
  // Extract phone number from JID (before @)
  return jid.split('@')[0] || 'unknown';
}
```
