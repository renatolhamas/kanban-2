# Story 5.1.4: Corrigir Mapeamento de Status CONNECTION_UPDATE no Webhook

> [!IMPORTANT]
> Esta story resolve um bug crítico de integração: o Evolution GO envia eventos `CONNECTION_UPDATE` com status `'open'`, mas o banco de dados (`tenants.connection_status`) possui uma check constraint que aceita apenas `'connected'`, `'connecting'`, `'disconnected'`, `'error'`, `'pending'`. Como o status enviado não é normalizado, a atualização do banco falha por violação de constraint, deixando o status de conexão desatualizado.

## Executor Assignment

executor: "@dev"
quality_gate: "@architect"
quality_gate_tools: ["lint", "typecheck"]

## Story Statement

**As a** system,
**I want to** normalize the connection status received from Evolution GO webhooks before saving it to the database,
**so that** the `tenants` table is updated correctly and the system reflects the actual state of the WhatsApp connection without database errors.

---

## Scope

**IN:**
- Adicionar mapeamento de status (Status Map) na função `handleConnectionUpdate` em `app/api/webhooks/evo-go/route.ts`.
- Mapear `'open'` (Evo GO) para `'connected'` (DB).
- Mapear `'close'` ou `'closed'` (Evo GO) para `'disconnected'` (DB).
- Mapear `'connecting'` (Evo GO) para `'connecting'` (DB).
- Mapear `'created'` (Evo GO) para `'pending'` (DB).
- Definir fallback para `'error'` (DB) para qualquer status desconhecido.
- Adicionar logs estruturados do mapeamento para facilitar debugging.
- Adicionar comentários documentando o mapeamento.

**OUT:**
- Mudanças no schema do banco de dados.
- Alteração em outros handlers de webhook além de `connection.update`.

## Dev Notes

### F5.plan.007 — Erro de Constraint no CONNECTION_UPDATE

**Problema Identificado:**
O webhook recebe `status: 'open'`, mas o banco espera `connection_status: 'connected'`.
Erro: `new row for relation "tenants" violates check constraint "tenants_connection_status_check"`

**Mapeamento Requerido:**

| Evolution GO Status | DB connection_status |
|---------------------|----------------------|
| `'open'`            | `'connected'`        |
| `'connecting'`      | `'connecting'`       |
| `'close'` / `'closed'` | `'disconnected'`  |
| `'created'`         | `'pending'`          |
| (unknown)           | `'error'`            |

**Localização:**
- Arquivo: `app/api/webhooks/evo-go/route.ts`
- Função: `handleConnectionUpdate()`

**Referência Técnica:**
[Source: docs/architecture/schema.md#81] - Embora haja discrepâncias nominais na documentação, o erro em tempo de execução confirma a existência da constraint e os valores esperados pelo banco conforme F5.plan.007.md.

### Arquivos a Modificar

1. **`app/api/webhooks/evo-go/route.ts`** — Modificar a função `handleConnectionUpdate` para incluir o mapeamento.
