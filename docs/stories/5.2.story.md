## Story Statement

**As a** system,  
**I want to** automatically route conversations to the Main kanban's first column,  
**so that** new contacts and re-engaged customers appear on the board without attendant intervention.

---

## Scope

**IN:**
- Auto-create conversations in Main kanban's first column for new contacts (no existing conversation)
- Re-open archived/closed conversations when the contact sends a message
- Move archived conversation to Main kanban's first column and set status to `active`
- Increment `unread_count` by 1 for each incoming message
- Update `last_message_at` and `evolution_message_id` on every message
- Route group messages to the same conversation as the individual contact
- Integrate into the MESSAGE webhook event handler

**OUT:**
- Manual conversation creation UI (handled separately)
- Conversation editing or manipulation UI
- Message content parsing or filtering
- Contact profile management
- Authentication or permission logic

## Risks

| Risk | Probability | Impact | Mitigation |
|------|-------------|--------|-----------|
| No "Main" kanban exists for tenant | Medium | Conversation creation fails silently | Error logging + tests verify Main kanban exists |
| Race condition: 2 webhooks for same contact simultaneously | Medium | Duplicate conversations or lost updates | Use database transactions + unique constraints |
| Kanban/column lookup returns null | Low | CREATE/UPDATE fails | Pre-create Main kanban on tenant creation |
| `unread_count` becomes out of sync | Low | Incorrect count display | Implement idempotent increment (check webhook ID before increment) |
| Archived conversation not found when re-opening | Low | New duplicate conversation created | Query includes archived status: `WHERE status IN ('active', 'archived')` |

### Webhook Event Reference

**Evo GO MESSAGE Event:**
- Event name: `MESSAGE` (reference: docs/evogo/github-evo-go/docs/wiki/recursos-avancados/events-system.md)
- Payload structure:
```json
{
  "event": "MESSAGE",
  "instance": "instance-name",
  "data": {
    "key": {
      "remoteJid": "5511999999999@s.whatsapp.net",
      "fromMe": false,
      "id": "3EB0C5A277F7F9B6C599"
    },
    "message": {
      "conversation": "Mensagem de texto"
    },
    "messageTimestamp": "1699999999",
    "pushName": "João Silva"
  }
}
```

### Implementation Strategy

```
MESSAGE event received
  ↓
Extract: tenantId, contact.id, conversation state
  ↓
Look up Main Kanban (is_main=true) + first column (order_position ASC)
  ↓
Check: Does conversation exist?
  ├─ NO: CREATE in Main/FirstCol, status=active, unread_count=1
  ├─ YES, status=active: UPDATE unread_count++, last_message_at
  └─ YES, status=archived/closed: UPDATE status=active, move to Main/FirstCol, unread_count=1
  ↓
RETURN success (webhook always 200)
```
