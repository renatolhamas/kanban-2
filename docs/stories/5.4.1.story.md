## 📝 Implementation Notes (Dev Notes)

### 🔴 CRITICAL GOTCHAS — DO NOT IGNORE

**If you ignore ANY of these, expect 4-8 hours of debugging:**

#### **Gotcha #1: Do NOT Pass tenant_id Manually**
```typescript
// ❌ WRONG - This creates a security vulnerability
const data = await supabase.rpc('get_conversations_with_last_message', {
  p_kanban_id: kanbanId,
  p_tenant_id: userInput.tenantId,  // ← NEVER DO THIS
});

// ✅ CORRECT - tenant_id comes from JWT, injected by custom_access_token_hook
const data = await supabase.rpc('get_conversations_with_last_message', {
  p_kanban_id: kanbanId,
  // p_tenant_id is NOT passed — RLS gets it from JWT
});
```
**Why:** Client can modify `p_tenant_id` and see another tenant's data. Security vulnerability.  
**Check:** Manual review should catch `p_tenant_id` in RPC calls.

---

#### **Gotcha #2: Always Include JWT in Authorization Header**
```typescript
// ❌ WRONG - No JWT = RLS blocks everything
const response = await fetch('/api/conversations?kanban_id=x');
// Returns 401 or empty array

// ✅ CORRECT - JWT in header
const token = localStorage.getItem('sb-jwt-token');
const response = await fetch('/api/conversations?kanban_id=x', {
  headers: {
    Authorization: `Bearer ${token}`,  // ← MANDATORY
  },
});
```
**Why:** RLS policies read `auth.jwt()` from Authorization header. Without JWT, RLS returns nothing.  
**Check:** Manual review should catch missing Authorization headers in all requests.

---

#### **Gotcha #3: Real-time Updates MUST Be Debounced (≥500ms)**
```typescript
// ❌ WRONG - 10 messages = 10 API calls = spam
channel.on('postgres_changes', {...}, (payload) => {
  fetchConversations();  // Immediate refetch
});

// ✅ CORRECT - Debounce prevents API spam
const debouncedFetch = debounce(() => fetchConversations(), 500);
channel.on('postgres_changes', {...}, (payload) => {
  debouncedFetch();  // Max once per 500ms
});
```
**Why:** Without debounce, 10 messages arriving in 100ms = 10 API calls = potential crash.  
**Check:** Performance test validates debounce working.

---

#### **Gotcha #4: Pagination LIMIT is Mandatory (if >100 conversations)**
```typescript
// ❌ WRONG - Full table scan on large datasets
const { data } = await supabase.rpc('get_conversations_with_last_message', {
  p_kanban_id: kanbanId,
});

// ✅ CORRECT - Limit results
const { data } = await supabase.rpc('get_conversations_with_last_message', {
  p_kanban_id: kanbanId,
})
  .range(0, 49);  // LIMIT 50
```
**Why:** Pagination prevents timeouts and excessive memory usage on large result sets.  
**Check:** RLS isolation test includes > 100 conversation scenario.
