## 📋 Story Summary

Restore the WhatsApp message persistence and Kanban dashboard functionality. 
1.  **Fix Webhook:** Update the Evolution GO webhook to use the correct column name `evolution_instance_id` (instead of the non-existent `evolution_instance_name`) to resolve the tenant correctly.
2.  **Restore RPC:** Ensure the `get_conversations_with_last_message` RPC is present in the database and properly documented.

Without these fixes, the system is "blind": webhooks return 200 but save nothing, and the dashboard remains empty.

---

## 🛠️ Technical Decisions (SSOT)

1.  **Code vs DB:** We will update the **code** to match the current database schema (`evolution_instance_id`). We will NOT rename the database column back.
2.  **RPC Stability:** The RPC `get_conversations_with_last_message` must use strict JWT tenant isolation (already planned in F5.plan.011).
3.  **Documentation:** `docs/db/functions.md` is the Single Source of Truth for database functions. It MUST be updated.

## 📝 Implementation Notes (Dev Notes)

### 🔴 CRITICAL GOTCHAS

#### **Gotcha #1: Silent Webhook Failures**
The Supabase client does not throw an error if you filter by a non-existent column (PostgREST simply returns 0 rows). This is why the webhook was returning 200 but doing nothing. 
**Verification:** Use `console.log` or temporary debug logs to ensure `tenantData` is NOT null during testing.

#### **Gotcha #2: RPC Tenant Isolation**
The RPC MUST extract the `tenant_id` from the JWT metadata using `((SELECT auth.jwt()) -> 'app_metadata' ->> 'tenant_id')::uuid`. Do NOT pass `p_tenant_id` as a parameter.
