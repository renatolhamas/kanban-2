# Story 5.4: Last Message Preview (SQL Subquery Optimization + UI Display)

## Story Statement

**As a** user viewing the Kanban board,  
**I want to** see a preview of the last message from each conversation on its card,  
**so that** I can quickly understand the latest activity in each chat without opening the modal.

---

## Scope

**IN:**
- Fetch last message content per conversation via optimized SQL subquery
- Display preview text truncated (max 100 chars) on conversation card
- Show message timestamp (relative: "2m ago", "1h ago", etc.)
- Differentiate message origin (incoming vs outgoing) with visual indicator
- Update preview in real-time when new messages arrive (via Supabase subscriptions)
- Handle NULL messages (conversations with no messages yet) gracefully

**OUT:**
- Media message preview display (videos, images, audio) — content extraction only
- Message search functionality — Story 7.x
- Message formatting/markdown rendering — Story 6.x
- Unread message count — Story 5.2.1 (already planned)

## Risks

| Risk | Probability | Impact | Mitigation |
|------|-------------|--------|-----------|
| N+1 query problem if not optimized with subquery | Medium | High performance degradation | Use correlated subquery or window function (ROW_NUMBER); test with EXPLAIN ANALYZE |
| Real-time subscription fires too frequently | Low | Memory/network waste | Subscribe only to messages for current tenant + conversation filter |
| Timestamp formatting library missing | Very Low | Component error | Use date-fns or Intl.RelativeTimeFormat (built-in) |
| Message content exceeds 100 chars display limit | Medium | Preview text truncates mid-word | Use `substring()` SQL function or text-overflow CSS with ellipsis |

### Selected Implementation Strategy (Analyst Decisions)

**SQL Optimization: Correlated Subquery**
The team selected the correlated subquery approach for its optimized performance in PostgreSQL when retrieving the "last 1" message per group.
```sql
SELECT 
  c.id,
  c.name,
  (SELECT content FROM messages 
   WHERE conversation_id = c.id 
   ORDER BY created_at DESC LIMIT 1) AS last_message,
  (SELECT created_at FROM messages 
   WHERE conversation_id = c.id 
   ORDER BY created_at DESC LIMIT 1) AS last_message_at,
  (SELECT sender_type FROM messages 
   WHERE conversation_id = c.id 
   ORDER BY created_at DESC LIMIT 1) AS last_sender_type,
  (SELECT media_url FROM messages 
   WHERE conversation_id = c.id 
   ORDER BY created_at DESC LIMIT 1) AS last_media_url
FROM conversations c
WHERE c.tenant_id = $1
ORDER BY last_message_at DESC;
```
*Note: Added `last_media_url` to support contextual placeholders.*

**Performance Notes:**
- Test both with `EXPLAIN ANALYZE` on 100+ conversations
- Verify index exists: `CREATE INDEX idx_messages_conversation_created ON messages(conversation_id, created_at DESC)`
- Benchmark: target ≤ 200ms for full Kanban load (Actual: 2.7ms verified in F5.plan.008)

### Frontend Implementation Points

**API Enhancement** (`app/api/conversations/route.ts`):
- Add `last_message`, `last_message_at`, `last_sender_type` to response
- Current endpoint likely only returns conversation list; extend with subquery
- Response format example:
```json

  "id": "conv-123",

  "last_message": "Hey, how are you?",
  "last_message_at": "2026-04-23T14:32:00Z",
  "last_sender_type": "contact"
}
```

**Component Enhancement** (`app/(authenticated)/home/page.tsx` - Kanban Card):
- Add preview section below contact name.
- **Timing:** Use `date-fns/formatDistanceToNow` for relative timestamps ("2m ago").
- **Prefix logic:** if `last_sender_type === 'contact'` → show contact name; else → show "You:".
- **Truncation:** Implement **CSS `line-clamp: 2`** (or 3) to handle visual truncation gracefully with ellipsis. Fetch full text (max 200) from API to ensure sufficient content for the clamp.
- **Placeholders:** If `content` is empty and `media_url` exists, display contextual label: `📷 Foto`, `🎥 Vídeo`, `🎵 Áudio`, or `📄 Arquivo`.
- Add subtle border/background to differentiate preview from card title.

**Real-time Subscription** (React hook):
- **Pattern:** Use **Tenant-level subscription** on the `messages` table (filtered by `tenant_id`).
- Update local state/preview when a new message event arrives for a visible conversation.
- Debounce updates if rapid messages arrive to prevent UI jitter.
