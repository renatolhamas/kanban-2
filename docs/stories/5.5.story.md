## Story Statement

**As an** attendant,
**I want to** send messages (text and media) directly from the Kanban interface to WhatsApp,
**so that** I can communicate with my contacts in real-time without leaving the platform.

---

## Scope

**IN:**
- API Route `/api/messages/send` to handle outgoing messages.
- Integration with Evolution GO endpoints: `/message/sendText` and `/message/sendMedia`.
- Message persistence in `messages` table with `sender_type: 'user'`.
- Automatic update of `conversations.last_message_at`.
- Basic UI mechanism (e.g., Quick Reply modal) to trigger sending during this phase.
- Handling of multiple media types: `image`, `video`, `audio`, `document`.
- Error handling for API failures (timeout, invalid token, instance disconnected).

**OUT:**
- Message status tracking (Sent/Delivered/Read) â€” Story 5.6.
- Full Chat Modal UI â€” Story 6.1.
- Message history pagination â€” Story 6.2.

## Acceptance Criteria

1. [ ] Attendant can send a text message to a contact; message arrives on WhatsApp in < 3s.
2. [ ] Attendant can send media (image/video/audio); file is correctly transmitted and viewable on WhatsApp.
3. [ ] All sent messages are persisted in the `messages` table with `sender_type: 'user'`.
4. [ ] The `conversations.last_message_at` field is updated to the current timestamp for every sent message.
5. [ ] The API Route `/api/messages/send` validates the user's `tenant_id` via JWT and ensures they only send messages for their own tenant.
6. [ ] If the Evolution GO API returns an error (e.g., instance disconnected), the UI displays a clear error notification and the message is NOT persisted as "sent".
7. [ ] UI provides immediate feedback (loading state) during transmission.

### Technical Details

**API Payload (Evolution GO):**
- `sendText`: `{ "number": "55...", "text": "..." }`
- `sendMedia`: `{ "number": "55...", "media": "URL", "mediatype": "image|video|audio|document", "caption": "..." }`

**Persistence Logic:**
- After successful API call:
```sql
BEGIN;
  INSERT INTO messages (conversation_id, sender_type, content, media_url, media_type)
  VALUES ($1, 'user', $2, $3, $4);
  
  UPDATE conversations 
  SET last_message_at = NOW(), updated_at = NOW()
  WHERE id = $1;
COMMIT;
```
