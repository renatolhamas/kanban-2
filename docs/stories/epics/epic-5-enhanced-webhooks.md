## Objective

Implement technical refinements for automated contact registration and conversation management via WhatsApp webhooks (Evolution GO). This epic focuses on data integrity, smart name synchronization, and conversation lifecycle automation.

## Business Value

Ensures a zero-touch experience for the attendant by automatically identifying contacts and restoring archived conversations when customers re-engage, preventing "lost" messages in archived states.

---

## Scope

### In Scope
- Multi-device suffix stripping (JID parsing).
- Automated group identification and registration.
- Smart name synchronization (protect manual edits vs. pushName updates).
- Dynamic tenant resolution via payload instanceId.
- Conversation routing: Auto-reopening archived conversations on new incoming messages.
- Unread count increment on message ingestion.

### Out of Scope
- Manual contact creation UI (covered in Epic 4).
- Bulk message history import.

## Success Criteria

- [ ] Webhook endpoint correctly normalizes E.164 phone numbers (stripping suffixes).
- [ ] New contacts are registered automatically with `is_group` flag where applicable.
- [ ] Manual edits to contact names are preserved during `pushName` updates.
- [ ] Archived conversations automatically move to the first column and `active` status upon new messages.
- [ ] `unread_count` increments correctly for all incoming messages.
