## 🎨 Colors {#colors}

### System Architecture

Our color system is built on semantic tokens that map to functional roles:

| Token | Value | Use Case |
|-------|-------|----------|
| `--color-primary` | #10b981 | Primary CTAs, active states, highlights |
| `--color-secondary` | #1e40af | Secondary buttons, links, secondary actions |
| `--color-danger` | #ef4444 | Destructive actions, errors, alerts |
| `--color-success` | #22c55e | Confirmations, valid states, success messages |
| `--color-warning` | #f59e0b | Warnings, cautions, attention-needed states |

### Surface Tones

Create visual hierarchy and depth with surface colors:

| Token | Value | Purpose |
|-------|-------|---------|
| `--surface` | #f7f9fb | Primary page background |
| `--surface-low` | #f2f4f6 | Secondary backgrounds, hover states |
| `--surface-lowest` | #ffffff | Card backgrounds, overlays, modals |
| `--surface-high` | #e6e8ea | Elevated surfaces, focus states |
| `--on-surface` | #191c1e | High contrast text (9:1 ratio) |

### WCAG AA Compliance

All colors are validated for accessibility:

- **Text Colors:** Minimum 4.5:1 contrast ratio on white backgrounds
- **Large Text:** Minimum 3:1 contrast ratio (18px+)
- **Graphics:** Minimum 3:1 contrast ratio for UI components

View the **Colors** story for interactive contrast ratio validation and color blindness simulations.

## 📝 Typography {#typography}

### Font Family

**Primary:** Manrope (geometric, modern, friendly)  
**Fallback:** -apple-system, BlinkMacSystemFont, "Segoe UI", sans-serif

```css
--font-manrope: "Manrope", -apple-system, BlinkMacSystemFont, "Segoe UI", sans-serif;
```

### Font Sizes

Scale follows a consistent progression for readable hierarchy:

| Token | Size | Use Case |
|-------|------|----------|
| `--font-xs` | 0.75rem (12px) | Captions, helper text, badges |
| `--font-sm` | 0.875rem (14px) | Labels, secondary text, fine print |
| `--font-base` | 1rem (16px) | Body text, default, paragraphs |
| `--font-lg` | 1.125rem (18px) | Emphasized text, subheadings |
| `--font-xl` | 1.25rem (20px) | Section headings |
| `--font-2xl` | 1.5rem (24px) | Page headings |
| `--font-3xl` | 2rem (32px) | Major headings |
| `--font-4xl` | 2.5rem (40px) | Hero headings |

### Line Heights

Optimize readability with appropriate spacing between lines:

| Token | Value | Use Case |
|-------|-------|----------|
| `--line-tight` | 1.2 (120%) | Headings, dense information |
| `--line-normal` | 1.5 (150%) | Body text, standard reading |
| `--line-relaxed` | 1.75 (175%) | Accessible, highly readable text |

**Accessibility Note:** Use `line-relaxed` for body text with font-base or smaller to ensure WCAG AA readability standards.

View the **Typography** story for live examples of each scale.

## 📐 Spacing {#spacing}

### 8px Scale System

All spacing uses a base unit of 8px for consistency and scalability:

| Token | Value | Pixels | Use Case |
|-------|-------|--------|----------|
| `--spacing-0` | 0 | 0px | No spacing |
| `--spacing-1` | 0.25rem | 4px | Extra tight (rare) |
| `--spacing-2` | 0.5rem | 8px | Component padding, tight margins |
| `--spacing-3` | 0.75rem | 12px | Form field padding |
| `--spacing-4` | 1rem | 16px | Standard padding, base margins |
| `--spacing-5` | 1.25rem | 20px | Medium spacing |
| `--spacing-6` | 1.5rem | 24px | Card spacing, section margins |
| `--spacing-8` | 2rem | 32px | Section margins |
| `--spacing-12` | 3rem | 48px | Major section breaks |
| `--spacing-16` | 4rem | 64px | Page sections |

### Implementation

Use in components via CSS variables or Tailwind utilities:

```css
/* CSS Variables */
padding: var(--spacing-4);
margin-bottom: var(--spacing-6);

/* Tailwind */
<div className="p-4 mb-6">Content</div>
```

View the **Spacing** story for interactive grid visualization.

## ✨ Shadows {#shadows}

### Elevation System

Shadows create depth and visual hierarchy:

| Token | Value | Elevation | Use Case |
|-------|-------|-----------|----------|
| `--shadow-sm` | 0px 2px 8px rgba(10, 25, 47, 0.04) | 1 | Subtle elevation, hints |
| `--shadow-base` | 0px 4px 16px rgba(10, 25, 47, 0.06) | 2 | Standard elevation, modals, cards |
| `--shadow-md` | 0px 8px 24px rgba(10, 25, 47, 0.08) | 3 | Raised elements, floating buttons |
| `--shadow-lg` | 0px 12px 32px rgba(10, 25, 47, 0.12) | 4 | Floating dialogs, popovers |
| `--shadow-xl` | 0px 20px 48px rgba(10, 25, 47, 0.16) | 5 | Maximum elevation, hero overlays |

### Best Practices

- Use **sm** for subtle hover states and icons
- Use **base** for cards and standard modals
- Use **md/lg** for floating or overlay elements
- Use **xl** sparingly, only for maximum emphasis

View the **Shadows** story for visual elevation examples.

## 🎬 Animations {#animations}

### Duration Tokens

Consistent animation timing for micro-interactions:

| Token | Value | Use Case |
|-------|-------|----------|
| `--animation-duration-fast` | 150ms | Quick interactions, state changes |
| `--animation-duration-normal` | 200ms | Standard transitions, fades, slides |

### Animation Types

| Token | Duration | Purpose |
|-------|----------|---------|
| `--animation-fade` | 200ms | Opacity transitions |
| `--animation-scale` | 200ms | Size/zoom transitions |
| `--animation-slide` | 200ms | Position transitions |

### Implementation

```css
/* CSS */
transition: opacity 200ms ease-in-out;
animation: fadeIn 200ms ease-out;

/* Tailwind */
<div className="transition-opacity duration-200">Fade</div>
```

View the **Animations** story for interactive animation previews.

## ♿ Accessibility

### Standards Compliance

All design tokens follow **WCAG 2.1 Level AA** guidelines:

- **Color Contrast:** Minimum 4.5:1 for text, 3:1 for graphics
- **Typography:** Font sizes ≥12px, line heights ≥1.2
- **Motion:** All animations respect `prefers-reduced-motion`
- **Focus States:** Visible keyboard navigation indicators

### Color Blindness

Our colors are tested for visibility with:
- Deuteranopia (red-green, ~6% of population)
- Protanopia (red-green, ~1% of population)
- Tritanopia (blue-yellow, <0.5% of population)

View the **Colors** story to see color blindness simulations.

### Testing Tools

- **Axe DevTools:** Browser extension for accessibility audits
- **WAVE:** WebAIM accessibility checker
- **Color Contrast Checker:** Verify WCAG compliance

## 🔗 Integration Guides

### CSS Variables

All tokens are available as CSS custom properties:

```css
/* app/globals.css */
:root {
  --color-primary: #10b981;
  --font-base: 1rem;
  --spacing-4: 1rem;
  --shadow-base: 0px 4px 16px rgba(10, 25, 47, 0.06);
}
```

### Tailwind Configuration

Tokens are extended into Tailwind utilities:

```typescript
/* tailwind.config.ts */
extend: {
  colors: {
    "token-primary": "var(--color-primary)",
    "token-secondary": "var(--color-secondary)",
    // ... more colors
  },
  spacing: {
    // Tailwind's default scale maps to our 8px system
  }
}
```

### React Components

Use tokens in styled components:

```typescript
// Example Button component
<button
  style={{
    backgroundColor: 'var(--color-primary)',
    padding: 'var(--spacing-4)',
    fontSize: 'var(--font-base)',
    boxShadow: 'var(--shadow-base)',
  }}
>
  Click me
</button>
```
